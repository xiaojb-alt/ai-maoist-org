# 红蓝对抗（内部攻防机制）

## 核心理念

在系统内部始终维持一个"红军"——专门负责找茬、攻击、发现漏洞的 Agent。

> 不打仗的时候就练对抗。打仗的时候才不会输。

## 运作机制

### 蓝军（执行方）
正常执行任务的 Agent 集群。负责完成实际业务目标。

### 红军（攻防方）
独立运行的 Agent，职责包括：
- 对蓝军的输出进行压力测试
- 寻找逻辑漏洞和事实错误
- 模拟恶意输入测试系统韧性
- 发现 Agent 偷懒/糊弄行为

### 战史库
每次对抗的结果写入战史库：
- 红军的攻击手段
- 蓝军的防御策略
- 成功/失败模式分析
- 优化建议

## 应用场景

| 场景 | 红军任务 |
|------|---------|
| 代码生成 | 找 Bug、安全漏洞、边界情况 |
| 文档生成 | 查事实错误、逻辑矛盾 |
| 数据分析 | 质疑数据源、验证方法 |
| 决策建议 | 找出盲点、反驳假设 |

## 注意事项

- 红军不能影响蓝军的正常执行（异步对抗）
- 红军自身也需要被监督（防止红军乱报）
- 对抗结果要可量化，避免主观判断

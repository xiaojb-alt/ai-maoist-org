# 群众路线（反馈闭环系统）

## 核心思想

> 从群众中来，到群众中去。

在 AI 系统中，"群众"就是最终使用者——用户。Agent 的评估标准不应该由系统自身定义，而应该来自用户的实际感受。

## 三层反馈机制

### 第一层：即时反馈
用户在每次交互后对结果打分（简单 +1/-1）：
- 快速、低成本的反馈信号
- 用于实时调整 Agent 行为
- 发现明显异常

### 第二层：深度反馈
用户或专家对 Agent 输出进行详细评价：
- 指出具体的错误和改进方向
- 标注幻觉、遗漏、偏差
- 存入战史库作为训练数据

### 第三层：群众调研
定期汇总分析所有用户反馈，提炼系统性问题和趋势：
- 哪些类型的任务频繁出问题？
- 哪些 Agent 的幻觉率最高？
- 哪些环节最浪费 Token？

## 数据架构

```
用户反馈
  ↓
反馈收集层（标准化格式化）
  ↓
信号分级（紧急/普通/参考）
  ↓
存入战史库 + 触发改进流程
  ↓
战史学习 → 更新 Agent Prompt / 路由策略
```

## 注意事项

- 用户反馈有噪声——需要统计聚合而非逐条响应
- 群众路线不等同于"用户永远是对的"
- 反馈闭环的延迟要尽量低
